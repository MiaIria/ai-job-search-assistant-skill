# AI Job Search Assistant References

This folder documents the product and research methodology behind
`ai-job-search-assistant-skill`.

The main skill remains a five-step workflow. These references explain the
standards behind that workflow so the skill is easier to maintain, audit, and
defend in interviews.

## Files

| File | Purpose |
| --- | --- |
| `workflow-methodology.md` | Explains why the skill is split into five inspectable steps. |
| `fit-scoring-rubric.md` | Defines the 100-point candidate-to-role matching rubric. |
| `company-research-framework.md` | Defines the five dimensions for company deep research. |
| `source-quality-standard.md` | Defines source reliability, recency, and anti-fabrication rules. |
| `resume-optimization-boundaries.md` | Defines what resume optimization may and may not do. |

## Interview Summary

The skill is designed around one product insight:

> Job search research is not one task. It is a funnel: discover opportunities,
> score fit, research deeply, personalize applications, and produce an action
> plan.

The references make the funnel explainable:

1. why each step exists,
2. what gets filtered at each step,
3. how match scores are calculated,
4. how company facts should be sourced,
5. where resume optimization must stop to avoid fabrication.
